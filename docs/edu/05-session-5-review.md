# 5차: 복기 + 실전 연습

> 목표: 전체 교육 내용을 복습하고, 미니 해커톤 시뮬레이션으로 실전 감각을 키운다.

---

## 이론 (60분)

### 1. 전체 복습 퀴즈 (15분)

**O/X 퀴즈 (가볍게)**
1. 클로드 코드는 코드를 직접 작성하지 않아도 웹앱을 만들 수 있다 (O)
2. API 키는 코드 파일에 직접 넣어야 한다 (X -> .env.local)
3. 데이터(JSON)를 바꾸면 화면이 자동으로 바뀐다 (O)
4. System Prompt를 바꾸면 AI 응답 스타일이 달라진다 (O)
5. 컴포넌트는 하나의 페이지에서만 사용할 수 있다 (X -> 재사용 가능)

**주관식 퀴즈**
1. 프로젝트 실행 명령어는? (`npm run dev`)
2. API 키를 저장하는 파일 이름은? (`.env.local`)
3. 이르미의 5대 카테고리는? (물가, 고용, 자영업, 금융, 부동산)
4. 위기 등급 4단계는? (긴급, 주의, 관찰, 안전)

### 2. 효과적인 프롬프트 패턴 총정리 (20분)

**상황별 프롬프트 템플릿**

| 상황 | 프롬프트 패턴 |
|------|-------------|
| 구조 파악 | "[파일/폴더]의 구조를 설명해줘" |
| UI 추가 | "[위치]에 [컴포넌트]를 추가해줘. [요구사항]" |
| UI 수정 | "[컴포넌트]의 [속성]을 [값]으로 변경해줘" |
| 데이터 변경 | "[JSON 파일]의 [필드]를 [값]으로 변경해줘" |
| API 연결 | "[API Route]에서 실제 API를 호출하도록 변경해줘" |
| 에러 수정 | "[에러 메시지]가 발생해. 원인을 찾아 수정해줘" |
| 기능 추가 | "[기능]을 추가해줘. 요구사항은 [목록]" |
| 전체 점검 | "모든 페이지가 정상 동작하는지 확인해줘" |

**프롬프트 레벨업 팁**
1. 맥락을 주면 더 정확한 결과: "대시보드 페이지의 카테고리 리스크 바에서..."
2. 제약 조건 명시: "globals.css의 CSS 변수를 사용해서..."
3. 참고 자료 제시: "data/mock/signals.json과 같은 형식으로..."
4. 단계적 요청: 큰 작업은 작은 단계로 나눠서 요청

### 3. 디버깅 기본 (15분)

**에러가 나면 어떻게 하나?**
```
1단계: 에러 메시지 읽기 (빨간 글씨)
2단계: 에러 메시지를 클로드 코드에 그대로 전달
3단계: 클로드 코드의 수정 제안 확인
4단계: 수정 후 다시 테스트
```

**자주 발생하는 에러 유형**
| 에러 | 원인 | 프롬프트 |
|------|------|---------|
| "Module not found" | 파일 경로 오류 | "import 경로를 확인하고 수정해줘" |
| "undefined" 표시 | 데이터 필드 불일치 | "[필드]가 undefined야. 데이터 구조를 확인해줘" |
| 빈 화면 | 컴포넌트 에러 | "화면이 안 나와. 콘솔 에러를 확인하고 수정해줘" |
| API 오류 | 키/네트워크 문제 | "API 호출이 실패해. 원인을 찾아줘" |

**에러 메시지 전달 프롬프트**
```
다음 에러가 발생했어:
[에러 메시지 복사+붙여넣기]
원인을 분석하고 수정해줘.
```

### 4. 해커톤 당일 전략 (10분)

**역할 분담 논의**
- 역할 A: 데이터 담당 (JSON 파일 관리, 데이터 교체)
- 역할 B: UI 담당 (화면 수정, 디자인 조정)
- 역할 C: API 담당 (AI 연동, 분석 실행)
- 도우미: 각 역할에 1:1 매칭되어 기술 지원

**시간 배분 가이드**
```
0~30분:   환경 확인 + 데이터 파일 배치
30~90분:  데이터 연결 + 분석 실행
90~150분: UI 조정 + 추가 기능
150~180분: 테스트 + 발표 준비
```

## 실습 (120분)

### 실습 1: 종합 실습 - 새 기능 구현 (60분)

> 각자 하나의 기능을 처음부터 끝까지 프롬프트로 구현

**미션 A: 대시보드에 "주간 변화" 위젯 추가**
```
대시보드에 "주간 변화" 위젯을 추가해줘.
- 종합 점수의 전주 대비 변화량 표시 (예: +3점)
- 상승이면 빨간색 화살표, 하락이면 초록색 화살표
- 5대 카테고리별 변화량도 작은 글씨로 표시
- globals.css의 색상 변수 사용
```

**미션 B: 위기 신호에 "최신순/위험순" 정렬 추가**
```
위기 신호 탭에 정렬 기능을 추가해줘.
- "최신순"과 "위험도순" 두 가지 정렬 옵션
- 버튼 또는 드롭다운으로 전환 가능
- 현재 선택된 정렬 기준이 강조 표시되게
```

**미션 C: 뉴스 분석에 "위험도 필터" 추가**
```
뉴스 분석 탭에 위험도 필터를 추가해줘.
- 긴급/주의/관찰/안전 4단계 필터 버튼
- 여러 등급을 동시에 선택 가능
- 선택된 등급의 뉴스만 표시
- 각 등급 버튼에 해당 등급 색상 적용
```

> 각자 미션 하나를 선택하고, 60분 내에 완성 목표
> 막히면 도우미와 함께 프롬프트를 다듬어서 진행

### 실습 2: 미니 해커톤 시뮬레이션 (30분)

**규칙**
- 30분 시간 제한
- 주어진 미션을 클로드 코드로 해결
- 도우미는 최소한의 힌트만 제공

**미션: "긴급 알림 배너" 구현**
```
조건:
- 대시보드 상단에 긴급 알림 배너 추가
- 위기 점수가 80 이상인 카테고리가 있으면 표시
- 해당 카테고리명과 점수 표시
- 닫기 버튼 포함
- 빨간색 계열 배경 (--danger 변수 사용)
```

**진행 순서**
1. 요구사항 분석 (5분)
2. 프롬프트 작성 + 실행 (15분)
3. 테스트 + 수정 (10분)

### 실습 3: 트러블슈팅 연습 (15분)

**의도적 에러 시나리오**

> 도우미가 사전에 코드에 작은 에러를 심어둠

시나리오 예시:
- JSON 파일에서 필드명 하나를 오타로 변경
- 컴포넌트에서 import 경로를 잘못 설정
- API Route에서 응답 형식을 변경

```
[탭]에서 에러가 발생하고 있어.
에러 메시지를 확인하고 원인을 찾아 수정해줘.
```

- 에러 메시지를 읽는 연습
- 클로드 코드에 에러를 전달하는 연습
- 수정 후 정상 동작 확인

### 실습 4: Q&A + 해커톤 최종 준비 (15분)

**자유 Q&A**
- 교육 과정에서 궁금했던 점

- 추가로 연습하고 싶은 부분

**해커톤 준비 최종 점검**

## 전체 교육 정리

### 회차별 핵심 키워드

| 회차 | 핵심 |
|------|------|
| 1차 | 환경설정, 터미널, 클로드 코드 첫 경험 |
| 2차 | 컴포넌트, UI 프롬프트, shadcn |
| 3차 | API, Claude AI, System Prompt, 환경변수 |
| 4차 | JSON 데이터, 필드 매핑, API Route 전환 |
| 5차 | 종합 실습, 디버깅, 해커톤 전략 |

### 기억해야 할 것

1. 프롬프트는 구체적으로, 한 번에 하나씩
2. 에러가 나면 메시지를 그대로 클로드 코드에 전달
3. 데이터(JSON)가 바뀌면 화면도 바뀐다
4. API 키는 .env.local에만 저장
5. 막히면 도우미에게 도움 요청
