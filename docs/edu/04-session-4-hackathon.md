# 4차: 해커톤 데이터 적용

> 목표: 현재 프로덕션의 목업 데이터를 실제 데이터로 교체하고, 해커톤 당일 작업 절차를 연습한다.

---

## 이론 (45분)

### 1. 목업 데이터 vs 실제 데이터 (10분)

**현재 상태 (목업)**
```
data/mock/dashboard.json  --> 미리 만들어둔 가짜 데이터
data/mock/signals.json    --> 예시 위기 신호
data/mock/news.json       --> 샘플 뉴스 기사
```

**해커톤 당일 (실제)**
```
data/news.json            --> 매경에서 제공하는 실제 뉴스 데이터
Claude API 분석            --> 실제 AI가 분석한 결과
```

**핵심**: 데이터 형식(구조)만 맞으면 화면은 자동으로 바뀐다

### 2. JSON 데이터 구조 (20분)

> 상세 내용은 [ref-data-structure.md](./ref-data-structure.md) 참조

**뉴스 기사 1건의 구조**
```json
{
  "id": "news-001",
  "title": "배달앱 수수료 또 인상...자영업자 '한숨'",
  "summary": "주요 배달앱들이 수수료를 최대 5%p 인상...",
  "category": "selfEmployed",
  "categoryLabel": "자영업",
  "keywords": ["배달앱", "수수료", "자영업자"],
  "publishedAt": "2026-01-19T09:00:00+09:00",
  "section": "경제"
}
```

**위기 신호 1건의 구조**
```json
{
  "id": "sig-001",
  "title": "배달앱 수수료 인상에 따른 자영업 위기",
  "severity": "critical",
  "score": 85,
  "category": "selfEmployed",
  "region": "전국",
  "evidence": ["배달앱 3사 수수료 평균 5%p 인상", "..."],
  "analysis": {
    "cause": "플랫폼 독점에 따른 수수료 인상 구조",
    "impact": "영세 자영업자 수익률 악화",
    "actionPoints": ["수수료 상한제 논의", "..."]
  }
}
```

### 3. 데이터 교체 절차 (10분)

```
1단계: 새 데이터 파일 받기 (해커톤 당일 제공)
2단계: data/ 폴더에 파일 배치
3단계: 데이터 필드 확인 (구조가 다르면 매핑 수정)
4단계: 개발 서버 재시작
5단계: 브라우저에서 3개 탭 전체 확인
6단계: 에러 있으면 클로드 코드로 수정
```

### 4. 해커톤 당일 체크리스트 (5분)

**시작 전 확인**
- [ ] 노트북 충전 100%
- [ ] 인터넷 연결 확인
- [ ] Node.js / Claude Code 동작 확인
- [ ] .env.local API 키 확인
- [ ] 이르미 프로젝트 최신 버전 pull

**작업 순서**
1. 뉴스 데이터 JSON 파일 배치
2. Claude API 연결 확인
3. 뉴스 분석 실행
4. 대시보드 데이터 갱신
5. 전체 동작 테스트
6. UI 미세 조정

## 실습 (135분)

### 실습 1: 현재 목업 데이터 분석 (20분)

**클로드 코드로 데이터 구조 파악**
```
data/mock 폴더에 있는 모든 JSON 파일 목록을 보여주고,
각 파일의 역할과 데이터 건수를 알려줘.
```

```
data/mock/dashboard.json의 구조를 분석해서
각 필드가 화면의 어느 부분에 표시되는지 설명해줘.
```

```
data/mock/signals.json에서 첫 번째 신호의 구조를 보여주고,
각 필드의 의미를 한글로 설명해줘.
```

### 실습 2: 데이터 교체 시뮬레이션 (30분)

**시나리오: 새로운 뉴스 데이터가 제공되었다고 가정**

**1단계: 테스트용 뉴스 데이터 생성**
```
data/mock/news.json을 참고해서
새로운 테스트 뉴스 기사 3건을 만들어줘.
카테고리는 물가, 고용, 금융으로 하나씩,
오늘 날짜 기준으로 만들어줘.
```

**2단계: 데이터 교체**
```
방금 만든 테스트 뉴스 데이터를 data/mock/news.json에 추가해줘.
기존 데이터는 유지하고 맨 앞에 추가해줘.
```

**3단계: 화면 확인**
- 뉴스 분석 탭에서 새 기사가 표시되는지 확인
- 카테고리 필터로 각 카테고리별 확인

### 실습 3: 필드 매핑 수정 연습 (25분)

**시나리오: 실제 데이터의 필드명이 다른 경우**

```
만약 실제 뉴스 데이터의 필드명이 다음과 같다면:
- "title" 대신 "headline"
- "publishedAt" 대신 "pub_date"
- "category" 대신 "section_code"

이 데이터를 우리 시스템에 맞게 변환하는
함수를 추가해줘. lib/api/mock-data.ts에
변환 함수를 만들어줘.
```

- 변환 후 데이터가 정상 표시되는지 확인
- 이런 상황이 해커톤 당일에 발생할 수 있음을 이해

### 실습 4: API Route 전환 연습 (30분)

**1단계: 현재 API 동작 확인**
```
app/api/news/route.ts를 열어서
현재 어떻게 데이터를 반환하는지 설명해줘.
```

**2단계: 데이터 소스 변경**
```
app/api/news/route.ts에서
현재 data/mock/news.json을 읽는 부분을
data/news.json(프로젝트 루트)을 읽도록 변경해줘.
파일이 없으면 목업 데이터를 사용하도록 fallback도 추가해줘.
```

**3단계: 테스트**
- data/ 폴더에 실제 데이터 파일이 있을 때 / 없을 때 모두 테스트
- 양쪽 모두 에러 없이 동작하는지 확인

### 실습 5: 대시보드 데이터 갱신 (15분)

**대시보드 점수 변경 테스트**
```
data/mock/dashboard.json의 종합 점수를
67에서 75로 변경해줘.
카테고리별 점수도 각각 5점씩 올려줘.
```

- 대시보드 탭에서 게이지, 바, 통계가 바뀌는지 확인
- "데이터가 바뀌면 화면이 자동으로 바뀐다"는 것을 직접 체험

### 실습 6: 전체 동작 테스트 (15분)

**3개 탭 전체 점검**

```
3개 탭(대시보드, 위기신호, 뉴스분석)을 모두 확인하고,
데이터가 정상적으로 표시되는지 점검해줘.
에러가 있으면 원인을 찾아 수정해줘.
```

- 대시보드: 종합 점수, 카테고리별 위험도, AI 브리핑
- 위기 신호: 신호 목록, 필터, 지역 지도
- 뉴스 분석: 기사 목록, 검색, 카테고리 필터

## 핵심 프롬프트 패턴 정리

### 데이터 관련
```
"data/mock/[파일명]의 구조를 분석해줘"
"[데이터]를 [형식]에 맞게 변환해줘"
"JSON 파일의 [필드]를 [값]으로 변경해줘"
```

### API Route 관련
```
"[API Route]의 데이터 소스를 [경로]로 변경해줘"
"파일이 없을 때 대비한 fallback을 추가해줘"
"API 응답에 에러 처리를 추가해줘"
```

### 디버깅 관련
```
"[탭]에서 데이터가 표시되지 않아. 원인을 찾아줘"
"[에러 메시지]가 발생해. 해결해줘"
"[컴포넌트]에서 [필드]가 undefined로 나와. 수정해줘"
```

## 도우미 가이드

- JSON 파일 수정 시 문법 에러(콤마, 괄호) 주의 -> 클로드 코드가 잡아주지만 확인
- "데이터 바꿨는데 안 바뀌어요" -> 개발 서버 재시작 또는 브라우저 새로고침 안내
- 필드 매핑 실습은 해커톤 당일 발생할 가능성이 높으므로 충분히 연습
- 실습 3~4에서 시간이 부족하면 핵심(데이터 교체)에 집중
