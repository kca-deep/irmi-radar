# 2차: UI 구현 - 프롬프트로 화면 만들기

> 목표: 이르미 프로젝트의 구조를 이해하고, 프롬프트로 UI를 수정/추가하는 방법을 익힌다.

---

## 이론 (60분)

### 1. 컴포넌트란? (15분)

**레고 블록 비유**
- 웹 화면은 작은 조각(컴포넌트)들을 조립해서 만든다
- 버튼, 카드, 목록, 헤더 등이 각각 하나의 컴포넌트
- 같은 컴포넌트를 여러 곳에서 재사용할 수 있다

**이르미 프로젝트의 컴포넌트 예시**
```
대시보드 페이지
  +-- 헤더 (로고 + 탭 메뉴)
  +-- AI 브리핑 패널 (요약 텍스트)
  +-- 위기 연쇄 패널 (그래프 + 신호 목록)
  +-- 카테고리 리스크 목록 (5개 카테고리 카드)
  +-- 정부 정책 패널
  +-- 국회 동향 패널
```

### 2. 이르미 프로젝트 3개 탭 (20분)

**탭 1: 대시보드 (메인)**
- 종합 리스크 점수 (0~100)
- 5대 카테고리별 위험도 (물가/고용/자영업/금융/부동산)
- AI가 생성한 오늘의 브리핑
- 위기 연쇄 관계도

**탭 2: 위기 신호**
- 감지된 위기 신호 목록 (긴급/주의/관찰/안전)
- 지역별 지도 (17개 시도)
- 카테고리/지역/등급 필터
- 관련 정부 지원 정책

**탭 3: 뉴스 분석**
- 매경 뉴스 기사 목록
- AI 분석 결과 (위험도 점수, 키워드)
- 카테고리별 분류
- 키워드 검색

### 3. 데이터가 화면에 표시되는 흐름 (15분)

```
JSON 데이터 파일          컴포넌트              화면
(data/mock/*.json)  -->  (components/*.tsx)  -->  (브라우저)

예시:
dashboard.json의         RiskGauge 컴포넌트가    원형 게이지로
overallScore: 67    -->  숫자를 받아서      -->  화면에 표시
```

**핵심 개념**: 데이터(JSON)가 바뀌면 화면도 자동으로 바뀐다

### 4. UI 프롬프트 작성법 (10분)

**단계별 접근**
```
1단계 (레이아웃): "카드를 3개 가로로 나열해줘"
2단계 (컴포넌트): "각 카드에 제목, 점수, 상태를 표시해줘"
3단계 (스타일):   "카드 배경을 등급별로 다른 색상으로 해줘"
```

**위치 지정 프롬프트**
```
"대시보드 페이지의 카테고리 리스크 목록 아래에 새로운 섹션을 추가해줘"
"헤더의 오른쪽에 알림 아이콘을 추가해줘"
```

## 실습 (120분)

### 실습 1: 이르미 프로젝트 클론 + 실행 (20분)

**1단계: 프로젝트 가져오기**
```bash
git clone [저장소 URL]
cd irmi-radar
```

**2단계: 의존성 설치 + 실행**
```bash
npm install
npm run dev
```

**3단계: 브라우저에서 확인**
- `http://localhost:3000` 접속
- 대시보드 / 위기 신호 / 뉴스 분석 탭 둘러보기
- 각 탭에서 어떤 정보가 표시되는지 파악

### 실습 2: 프로젝트 구조 탐색 (15분)

**클로드 코드로 구조 파악하기**
```
프로젝트의 폴더 구조를 간단하게 설명해줘.
특히 components 폴더 안에 어떤 컴포넌트들이 있는지 알려줘.
```

```
대시보드 페이지가 어떤 컴포넌트들로 구성되어 있는지
목록으로 설명해줘.
```

### 실습 3: 간단한 UI 수정 (30분)

**도전 과제 A: 텍스트 수정**
```
대시보드 헤더의 부제목을 "실시간 민생 동향 모니터링"으로 변경해줘.
```

**도전 과제 B: 레이아웃 변경**
```
카테고리 리스크 목록에서 카드의 배치를 변경해줘.
현재 세로 목록 형태를 2열 그리드로 바꿔줘.
```

**도전 과제 C: 스타일 조정**
```
AI 브리핑 패널의 테두리를 좀 더 두껍게 하고,
패널 안의 여백을 넓혀줘.
```

### 실습 4: 새 컴포넌트 추가 (35분)

**미션: "오늘의 한줄 요약" 카드 추가**

```
대시보드 페이지 상단에 "오늘의 한줄 요약" 카드를 추가해줘.
- 카드 안에 "민생 위기 종합 점수가 67점으로, 주의 단계입니다." 라는 텍스트 표시
- 카드 배경은 약간 노란색 계열
- 왼쪽에 경고 아이콘 표시
- 글로벌 CSS 변수를 사용해서 색상을 지정해줘
```

- 결과 확인 후 추가 수정 프롬프트로 다듬기
- "텍스트 크기를 좀 더 키워줘"
- "카드 모서리를 둥글게 해줘"

### 실습 5: shadcn 컴포넌트 활용 (20분)

**클로드 코드에게 shadcn 활용 요청하기**
```
shadcn의 Badge 컴포넌트를 사용해서
카테고리 이름 옆에 등급 배지를 추가해줘.
긴급은 빨강, 주의는 주황, 관찰은 노랑, 안전은 초록으로 표시.
```

```
shadcn의 Progress 컴포넌트를 활용해서
카테고리별 위험도를 프로그레스 바로 표시해줘.
```

## 핵심 프롬프트 패턴 정리

### 구조 파악용
```
"이 프로젝트의 폴더 구조를 설명해줘"
"[파일명]이 어떤 역할을 하는지 설명해줘"
"대시보드 페이지가 어떤 컴포넌트들로 구성되어 있어?"
```

### UI 수정용
```
"[컴포넌트명]의 [속성]을 [값]으로 변경해줘"
"[위치]에 [컴포넌트]를 추가해줘"
"[컴포넌트]의 레이아웃을 [방식]으로 변경해줘"
```

### 스타일 수정용
```
"[컴포넌트]의 배경색/글자색/여백/테두리를 변경해줘"
"반응형으로 만들어서 모바일에서도 잘 보이게 해줘"
"다크 모드에서도 잘 보이게 해줘"
```

## 도우미 가이드

- 클론/설치 과정에서 에러 발생 시 직접 해결 지원
- 프롬프트 결과가 의도와 다를 때: "어떻게 고쳐달라고 하면 좋을까요?" 식으로 유도
- 파일 구조 질문이 나오면 실제 폴더를 열어 보여주며 설명
- "이 코드가 뭔가요?" -> "클로드가 만든 코드인데, 우리는 내용보다 결과에 집중합시다"
