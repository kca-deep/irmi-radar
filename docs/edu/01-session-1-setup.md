# 1차: 환경설정 + 클로드 코드 & MCP 체험

> 목표: 클로드 코드와 MCP 서버의 개념을 이해하고, 프롬프트만으로 웹앱을 만드는 경험을 한다.

---

## 이론 (60분)

### 1. 웹앱이란? (10분)

**일반 앱 vs 웹앱**
- 일반 앱: 스토어에서 설치 (카카오톡, 네이버 등)
- 웹앱: 브라우저에서 바로 사용 (설치 불필요)
- 우리가 만들 것: 브라우저에서 동작하는 "민생위기 대시보드"

**웹앱의 구성 요소**
- HTML: 뼈대 (어떤 내용이 있는지)
- CSS: 옷 (어떻게 보이는지)
- JavaScript: 두뇌 (어떻게 동작하는지)
- 하지만 우리는 직접 작성하지 않는다 -> 클로드 코드가 대신 작성

### 2. 클로드 코드란? (15분)

**개념**
- AI가 코드를 대신 작성해주는 터미널 도구
- 말(프롬프트)로 지시하면 파일을 만들고 수정해준다
- 프로그래밍 언어를 몰라도 웹앱을 만들 수 있다

**동작 방식**
```
사용자 프롬프트 -> 클로드 코드가 분석 -> 코드 생성/수정 -> 결과 확인
```

**기본 명령어**
| 명령어 | 설명 |
|--------|------|
| `claude` | 클로드 코드 시작 |
| `/help` | 도움말 보기 |
| `/clear` | 대화 초기화 |
| `Ctrl+C` | 현재 작업 중단 |
| `Esc` | 입력 취소 |

### 3. MCP 서버란? (20분)

**MCP (Model Context Protocol) 개념**

```
비유: 클로드 코드 = 만능 요리사
      MCP 서버   = 전문 조리 도구 세트

요리사 혼자서도 요리할 수 있지만,
전문 도구(오븐, 믹서기, 에어프라이어)가 있으면
더 빠르고 정확하게 요리할 수 있다.
```

- MCP 서버 = 클로드 코드에 추가되는 "전문 도구"
- 각 MCP 서버는 특정 분야에 특화된 기능 제공
- 클로드 코드가 필요할 때 자동으로 적절한 MCP 서버를 호출

**우리 프로젝트에 설치된 4개 MCP 서버**

| MCP 서버 | 역할 | 비유 |
|----------|------|------|
| **shadcn** | UI 컴포넌트 관리 | 인테리어 카탈로그 (버튼, 카드 등 기성 UI 부품 제공) |
| **sequential-thinking** | 복잡한 문제 단계별 사고 | 전략 기획 노트 (복잡한 설계를 단계별로 정리) |
| **context7** | 최신 기술 문서 조회 | 백과사전 (라이브러리 사용법을 실시간 검색) |
| **playwright** | 브라우저 자동 테스트 | 테스트 로봇 (화면을 자동으로 클릭하고 확인) |

**MCP 서버가 언제 동작하나?**
```
직접 호출할 필요 없음! 클로드 코드가 알아서 판단

예시 1: "버튼 컴포넌트를 추가해줘"
  -> 클로드 코드가 자동으로 shadcn MCP 사용

예시 2: "리스크 점수 계산 로직을 설계해줘"
  -> 클로드 코드가 자동으로 sequential-thinking MCP 사용

예시 3: "Next.js App Router에서 페이지 라우팅 방법을 알려줘"
  -> 클로드 코드가 자동으로 context7 MCP 사용

예시 4: "메인 페이지가 정상적으로 보이는지 확인해줘"
  -> 클로드 코드가 자동으로 playwright MCP 사용
```

**각 MCP 서버 상세 설명**

#### shadcn MCP - UI 컴포넌트 관리
```
제공 기능:
- 사용 가능한 UI 컴포넌트 목록 검색
- 컴포넌트 사용 예시 코드 조회
- 컴포넌트 설치 명령어 생성
- 작업 후 체크리스트 검증

사용 예시:
"Card 컴포넌트를 검색해줘"
  -> shadcn MCP가 Card 컴포넌트 정보 + 사용 예시 제공

"Badge 컴포넌트를 프로젝트에 추가해줘"
  -> shadcn MCP가 설치 명령어 생성 + 자동 설치
```

#### sequential-thinking MCP - 단계별 사고
```
제공 기능:
- 복잡한 문제를 단계별로 분해
- 각 단계의 결론을 기반으로 다음 단계 진행
- 필요하면 이전 단계 재검토 (되돌아가기)
- 최종 결론 도출

사용 예시:
"위기 점수 계산 방식을 설계해줘"
  -> 단계 1: 입력 데이터 분석
  -> 단계 2: 카테고리별 가중치 설정
  -> 단계 3: 점수 산출 공식 설계
  -> 단계 4: 등급 분류 기준 정리
  -> 최종: 종합 설계안 도출
```

#### context7 MCP - 기술 문서 검색
```
제공 기능:
- 라이브러리/프레임워크의 최신 문서 조회
- 실제 코드 예시 검색
- 정확한 사용법 확인

사용 예시:
"Next.js에서 API Route 만드는 방법을 알려줘"
  -> context7 MCP가 Next.js 최신 공식 문서에서 검색
  -> 정확한 코드 예시 제공

"Tailwind CSS로 그리드 레이아웃 만드는 법"
  -> context7 MCP가 Tailwind 문서에서 검색
```

#### playwright MCP - 브라우저 테스트
```
제공 기능:
- 브라우저 자동 열기/닫기
- 특정 URL 접속
- 화면 스크린샷 촬영
- 버튼 클릭, 텍스트 입력 자동화
- 화면 요소 확인 (스냅샷)
- 콘솔 에러 확인

사용 예시:
"메인 페이지를 브라우저에서 열고 스크린샷을 찍어줘"
  -> playwright MCP가 브라우저 열기 -> 접속 -> 캡처

"로그인 버튼을 클릭하면 어떤 화면이 나오는지 확인해줘"
  -> playwright MCP가 자동으로 클릭 -> 결과 확인
```

### 4. 프롬프트 기본 원칙 (10분)

**나쁜 프롬프트 vs 좋은 프롬프트**

| 나쁜 예 | 좋은 예 |
|---------|---------|
| "페이지 만들어" | "메인 페이지에 제목 '안녕하세요'를 추가해줘" |
| "예쁘게 해줘" | "배경색을 파란색으로, 글자색을 흰색으로 바꿔줘" |
| "다 바꿔" | "헤더의 제목 텍스트를 '나의 소개'로 변경해줘" |

**프롬프트 작성 3원칙**
1. 구체적으로: 무엇을, 어디에, 어떻게 변경할지 명시
2. 한 번에 하나씩: 여러 변경을 한꺼번에 요청하지 않기
3. 맥락 제공: 현재 상태나 원하는 결과를 설명

### 5. 프로젝트 구조 맛보기 (5분)

```
my-app/                  <- 프로젝트 폴더 (집 전체)
  app/                   <- 페이지들이 모여 있는 곳 (방들)
    page.tsx             <- 메인 페이지 (거실)
    layout.tsx           <- 공통 틀 (집의 구조)
    globals.css          <- 전체 스타일 (인테리어)
  public/                <- 이미지 등 (장식품)
  package.json           <- 설정 파일 (집 설명서)
  .mcp.json              <- MCP 서버 설정 (전문 도구 목록)
```

- `app/page.tsx` -> 브라우저에서 열리는 첫 화면
- `app/layout.tsx` -> 모든 페이지에 공통으로 적용되는 틀
- `.mcp.json` -> 클로드 코드가 사용할 MCP 서버 목록

**4단계: 설치 확인**

### 실습 3: 자기소개 페이지 만들기 (25분)

**프롬프트 예시 (순서대로 입력)**

```
1. 메인 페이지의 내용을 모두 지우고,
   "OOO의 자기소개" 라는 제목을 가운데 정렬로 추가해줘.

2. 제목 아래에 다음 내용을 추가해줘:
   - 이름: OOO
   - 소속: OO팀

3. 페이지 배경색을 연한 파란색으로 변경해줘.
```

> 4개 MCP 서버를 실제로 사용해보는 시간

#### 4-1. shadcn MCP 체험 (10분)

**UI 컴포넌트 검색하기**
```
shadcn에서 사용 가능한 Card 컴포넌트를 검색해줘.
어떤 종류가 있는지 보여줘.
```

**컴포넌트 예시 확인하기**
```
shadcn Card 컴포넌트의 사용 예시를 보여줘.
```

**컴포넌트 설치 + 적용하기**
```
shadcn의 Badge 컴포넌트를 설치하고,

```

- 결과 확인: 카드에 배지가 추가되었는지 브라우저에서 확인
- 핵심 포인트: "우리가 직접 만들지 않아도 이미 만들어진 UI 부품이 있다"

#### 4-2. sequential-thinking MCP 체험 (10분)

**복잡한 문제를 단계별로 풀기**
```
자기소개 페이지를 포트폴리오 웹사이트로 확장하려면
어떤 페이지들이 필요하고, 각 페이지에 어떤 내용이 들어가야 할지
단계별로 설계해줘. sequential-thinking을 사용해줘.
```

- 관찰 포인트: 클로드가 여러 단계로 나눠서 사고하는 과정 확인
- "한번에 답하지 않고, 단계별로 생각을 정리하는 것"을 체험
- 실제 코드 작성은 하지 않아도 됨 (설계 결과만 확인)

#### 4-3. context7 MCP 체험 (10분)

**기술 문서 실시간 조회**
```
context7을 사용해서 Next.js App Router에서
새로운 페이지를 추가하는 방법을 알려줘.
최신 공식 문서 기준으로 설명해줘.
```

```
context7을 사용해서 Tailwind CSS에서
flexbox로 가로 정렬하는 방법을 찾아줘.
```

- 관찰 포인트: 인터넷 검색 없이 최신 문서를 바로 찾아주는 것을 체험
- "모르는 기술이 있어도 context7이 공식 문서를 찾아준다"

#### 4-4. playwright MCP 체험 (15분)

**브라우저 자동 제어 체험**

> 개발 서버가 실행 중이어야 합니다 (npm run dev)

```
playwright로 http://localhost:3000 페이지를 열고
현재 화면의 스크린샷을 찍어줘.
```

```
playwright로 현재 페이지의 접근성 스냅샷을 보여줘.
화면에 어떤 요소들이 있는지 목록으로 확인하고 싶어.
```

```
playwright로 현재 페이지에서
카드 요소를 클릭해보고, 클릭 후 상태를 확인해줘.
```

- 관찰 포인트: 사람이 직접 브라우저를 조작하지 않아도 자동으로 테스트
- "만든 페이지가 제대로 동작하는지 AI가 대신 확인해준다"

**MCP 서버 활용 종합 프롬프트**
```
내 자기소개 페이지를 점검해줘.
1. playwright로 페이지 스크린샷 찍어서 현재 상태 확인
2. 개선할 점이 있으면 shadcn 컴포넌트를 활용해서 수정
3. 수정 후 다시 playwright로 스크린샷 찍어서 비교
```

- 여러 MCP 서버가 함께 동작하는 것을 체험

## MCP 설정 파일 살펴보기 (참고)

이르미 프로젝트의 MCP 설정 파일 (`.mcp.json`):

```json
{
  "mcpServers": {
    "shadcn": {
      "command": "npx",
      "args": ["shadcn@latest", "mcp"]
    },
    "sequential-thinking": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-sequential-thinking"]
    },
    "context7": {
      "command": "npx",
      "args": ["-y", "@upstash/context7-mcp"]
    },
    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp@latest"]
    }
  }
}
```

- 이 파일이 프로젝트 루트에 있으면 클로드 코드 시작 시 자동으로 MCP 서버 로드
- 교육생이 직접 수정할 필요 없음 (이미 설정 완료)
- "이런 설정 파일이 있구나" 정도로 이해하면 충분

## 도우미 가이드

### 환경 설치
- 환경 설치에서 막히면 직접 해결해주되, 과정을 설명해준다
- 사전 설치가 가능하면 교육 전에 미리 설치하여 시간 절약

### 프롬프트 실습
- 프롬프트 입력은 반드시 교육생이 직접 하게 한다
- 에러가 나면 에러 메시지를 함께 읽고, 프롬프트로 해결을 시도하게 유도
- "이건 안 될 것 같은데요" -> "한번 해봅시다. 안 되면 프롬프트를 바꿔보면 됩니다"

### MCP 실습
- MCP는 클로드 코드가 자동으로 호출하므로, 교육생이 별도로 조작할 것은 없음
- 클로드 코드의 출력에서 "MCP 서버를 사용하고 있다"는 표시가 나오면 짚어주기
- shadcn 체험 시: 컴포넌트가 설치되는 과정을 함께 관찰
- playwright 체험 시: 브라우저가 자동으로 열리는 것을 함께 관찰
- sequential-thinking 체험 시: 단계별 사고 과정이 출력되는 것을 함께 읽기
- context7 체험 시: 검색 결과가 공식 문서 기반이라는 것을 강조
- 시간이 부족하면 shadcn + playwright 2개를 우선 체험 (가장 시각적)
