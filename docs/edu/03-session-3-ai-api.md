# 3차: AI API 활용

> 목표: Claude API의 개념을 이해하고, 프롬프트로 AI 기능을 웹앱에 연동하는 방법을 익힌다.

---

## 이론 (60분)

### 1. API란? (15분)

**음식점 비유**
```
손님(우리 웹앱)  -->  주문서(API 요청)  -->  주방(Claude AI)
                                              |
손님(결과 수신)  <--  음식(API 응답)    <--  주방(처리 완료)
```

- API = 프로그램끼리 대화하는 약속된 방법
- 우리 웹앱이 Claude에게 "이 뉴스 분석해줘"라고 요청
- Claude가 분석 결과를 돌려보내줌

**일상 속 API 예시**
- 날씨 앱: 기상청 API에 "서울 날씨 알려줘" 요청
- 지도 앱: 네이버 지도 API에 "길 찾아줘" 요청
- 이르미: Claude API에 "이 뉴스 위험도 분석해줘" 요청

### 2. Claude API 기본 개념 (20분)

**3가지 핵심 요소**

| 요소 | 역할 | 비유 |
|------|------|------|
| System Prompt | AI의 역할/성격 설정 | "당신은 민생 위기 분석 전문가입니다" |
| User Message | 사용자의 질문/요청 | "이 뉴스를 분석해줘" |
| Assistant Response | AI의 응답 | 분석 결과 (점수, 카테고리, 요약 등) |

**System Prompt의 중요성**
```
System Prompt를 바꾸면 AI의 응답 방식이 달라진다:

예시 1: "간결하게 한 줄로 답변해주세요"
  -> "자영업 위기 등급: 주의 (72점)"

예시 2: "상세하게 원인과 대책을 포함해서 답변해주세요"
  -> "자영업 분야 위기 등급은 72점(주의)입니다.
      주요 원인은 배달앱 수수료 인상(15%)과
      임대료 상승(전년 대비 8%)이며..."
```

### 3. 환경변수와 API 키 (10분)

**API 키란?**
- API를 사용하기 위한 "비밀 열쇠"
- 열쇠가 없으면 API 사용 불가
- 절대로 코드에 직접 작성하면 안 됨 (유출 위험)

**환경변수 파일 (.env.local)**
```bash
# 이 파일에 API 키를 저장
ANTHROPIC_API_KEY=sk-ant-xxxxx      # Claude AI
DATA_GO_KR_API_KEY=xxxxx            # 보조금24 (정부 지원정책)
ASSEMBLY_API_KEY=xxxxx              # 국회 오픈API
```

- `.env.local` 파일은 내 컴퓨터에만 존재 (Git에 올라가지 않음)
- 서버에서만 읽을 수 있음 (브라우저에서 접근 불가)

### 4. API Route 개념 (15분)

**왜 API Route가 필요한가?**
```
                    API 키 노출 위험!
브라우저(클라이언트) ----X----> Claude API

                    안전한 방법
브라우저 --> API Route(서버) --> Claude API
             (API 키 숨김)
```

**이르미의 API Route 목록**
| 경로 | 역할 |
|------|------|
| `/api/dashboard` | 대시보드 종합 데이터 조회 |
| `/api/signals` | 위기 신호 목록 조회 |
| `/api/news` | 뉴스 기사 목록 조회 |
| `/api/analyze` | Claude AI로 뉴스 분석 |
| `/api/chat` | AI 채팅 응답 생성 |
| `/api/briefing` | AI 브리핑 생성 |
| `/api/policies` | 보조금24 정부 지원정책 조회 |
| `/api/assembly` | 국회 입법/청원 동향 조회 |

## 실습 (120분)

### 실습 1: 환경변수 설정 (15분)

**1단계: .env.local 파일 확인**
```
프로젝트 루트의 .env.local 파일 내용을 보여줘.
API 키가 설정되어 있는지 확인해줘.
```

**2단계: API 키 입력**
```
.env.local 파일에 다음 환경변수를 추가해줘:
ANTHROPIC_API_KEY=[교육용 키]
```

> 도우미가 교육용 API 키 제공

### 실습 2: AI 채팅 기능 테스트 (25분)

**1단계: 채팅 패널 열기**
- 브라우저 우측 하단 채팅 버튼 클릭
- 추천 질문 중 하나 선택해서 테스트

**2단계: 채팅 응답 확인**
- 현재는 목업(미리 준비된) 응답이 나옴
- 실제 API로 전환하면 Claude가 직접 답변

**3단계: 클로드 코드로 실제 API 연결**
```
app/api/chat/route.ts 파일에서
현재 목업 응답 대신 실제 Claude API를 호출하도록 변경해줘.
시스템 프롬프트는 "당신은 민생 위기 분석 전문가입니다.
한국의 물가, 고용, 자영업, 금융, 부동산 분야의
위기 상황에 대해 전문적으로 답변해주세요."로 설정해줘.
```

**4단계: 실제 AI 응답 확인**
- 채팅에서 다양한 질문 테스트
  - "현재 물가 상황은 어때?"
  - "소상공인에게 가장 위험한 상황은?"
  - "부동산 전세 사기 대응 방법은?"

### 실습 3: System Prompt 튜닝 (20분)

**System Prompt 변경 실험**

```
chat API의 시스템 프롬프트를 변경해줘:
"당신은 친절한 민생 상담사입니다.
어려운 경제 용어를 쉽게 풀어서 설명하고,
구체적인 대응 방법을 3가지씩 제안해주세요.
답변은 300자 이내로 해주세요."
```

- 같은 질문을 다시 해보고 답변 차이 비교
- 답변 길이, 톤, 구체성이 어떻게 달라지는지 관찰

**다른 System Prompt 실험**
```
시스템 프롬프트를 다시 변경해줘:
"당신은 데이터 기반 위기 분석가입니다.
감정적 표현 대신 수치와 데이터를 중심으로 답변하세요.
각 분석에 신뢰도 점수(0~100)를 포함해주세요."
```

### 실습 4: 뉴스 분석 API 테스트 (25분)

**1단계: 뉴스 분석 탭에서 분석 실행**
- 뉴스 분석 탭 이동
- 분석 실행 버튼 클릭
- 분석 진행 과정 관찰 (7단계)

**2단계: 분석 API 구조 이해**
```
app/api/analyze/route.ts 파일의 구조를 설명해줘.
어떤 데이터를 받아서 어떤 결과를 돌려주는지 알려줘.
```

**3단계: 분석 프롬프트 수정**
```
뉴스 분석 시 Claude에게 전달하는 프롬프트에
다음 내용을 추가해줘:
"각 뉴스 기사의 영향을 받는 주요 지역을 함께 분석해주세요."
```

### 실습 5: 외부 API 데이터 확인 (20분)

**보조금24 API (정부 지원 정책)**
```
보조금24 API가 어떤 데이터를 가져오는지 확인해줘.
현재 "물가" 카테고리로 검색하면 어떤 정책들이 나오는지 보여줘.
```

**국회 오픈API (입법 동향)**
```
국회 API에서 "소상공인" 키워드로 검색하면
어떤 법안들이 나오는지 확인해줘.
```

### 실습 6: AI 응답 결과 비교 (15분)

**팀 활동: 프롬프트 대결**
- 같은 질문에 대해 각자 다른 System Prompt 작성
- 3명의 결과를 비교
- 가장 유용한 응답을 만든 프롬프트 선정

## 핵심 프롬프트 패턴 정리

### API 연결 관련
```
"[API Route 파일]에서 목업 대신 실제 API를 호출하도록 변경해줘"
"시스템 프롬프트를 [내용]으로 설정해줘"
"API 응답에 [필드]를 추가해줘"
```

### API 테스트 관련
```
"[API 경로]를 호출해서 결과를 보여줘"
"[카테고리]로 필터링해서 데이터를 가져와줘"
"API 응답이 올바른지 확인해줘"
```

### 프롬프트 엔지니어링
```
"Claude에게 전달하는 프롬프트를 [방향]으로 수정해줘"
"응답 형식을 [형식]으로 변경해줘"
"분석 깊이를 더 상세하게/간결하게 조정해줘"
```

## 도우미 가이드

- API 키는 교육 시작 전 미리 준비 (공유용 키)
- API 호출 실패 시: 네트워크 확인 -> API 키 확인 -> 프롬프트 확인
- "코드가 너무 어려워요" -> "코드 내용은 몰라도 됩니다. 프롬프트로 지시만 하면 돼요"
- System Prompt 실험 시 재미있는 예시를 준비해두면 좋음
  - "사투리로 답변해줘" 등 가벼운 실험도 OK
