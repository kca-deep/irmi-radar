# 클로드 코드 프롬프트 가이드

> 클로드 코드를 효과적으로 사용하기 위한 프롬프트 작성법을 안내합니다.

---

## 클로드 코드 기본 사용법

### 시작하기
```bash
# 프로젝트 폴더에서 클로드 코드 시작
cd irmi-radar
claude
```

### 기본 명령어

| 명령어 | 설명 | 언제 사용 |
|--------|------|----------|
| `claude` | 클로드 코드 시작 | 터미널에서 처음 시작할 때 |
| `/help` | 도움말 보기 | 사용법이 궁금할 때 |
| `/clear` | 대화 초기화 | 새로운 주제로 전환할 때 |
| `Ctrl+C` | 현재 작업 중단 | 작업이 너무 오래 걸릴 때 |
| `Esc` | 입력 취소 | 작성 중인 프롬프트를 지울 때 |

### 동작 방식
```
1. 프롬프트 입력 (한국어 OK)
2. 클로드가 분석 후 작업 계획 제시
3. 파일 수정/생성 허용 여부 확인
4. 작업 완료 후 결과 설명
5. 브라우저에서 결과 확인
```

## 프롬프트 작성 3원칙

### 원칙 1: 구체적으로 (What + Where + How)

```
나쁜 예:
  "페이지 수정해줘"

좋은 예:
  "대시보드 페이지의 종합 점수 게이지 아래에
   '전주 대비 +3점 상승' 텍스트를 추가해줘"
```

- 무엇을: 텍스트, 버튼, 카드, 색상 등
- 어디에: 어떤 페이지의 어떤 위치
- 어떻게: 구체적인 모양, 동작

### 원칙 2: 한 번에 하나씩

```
나쁜 예:
  "헤더 색상 바꾸고 버튼 추가하고 폰트도 변경하고
   새 페이지도 만들어줘"

좋은 예:
  1단계: "헤더의 배경색을 진한 남색으로 변경해줘"
  2단계: "헤더 오른쪽에 '알림' 버튼을 추가해줘"
  3단계: "헤더의 제목 폰트를 좀 더 크게 해줘"
```

- 작은 단위로 나눠서 요청
- 각 단계마다 결과를 확인
- 문제가 생기면 해당 단계만 수정

### 원칙 3: 맥락 제공

```
나쁜 예:
  "목록을 보여줘"

좋은 예:
  "위기 신호 탭의 신호 목록에서
   현재 카드 형태로 표시되는 것을
   테이블(표) 형태로 변경해줘.
   열은 제목, 등급, 점수, 카테고리, 날짜로 해줘."
```

- 현재 어떤 상태인지 설명
- 원하는 결과가 무엇인지 설명
- 참고할 만한 기존 요소가 있으면 언급

## 상황별 프롬프트 템플릿

### 1. 프로젝트 구조 파악

```
이 프로젝트의 폴더 구조를 설명해줘.
```

```
components/dashboard 폴더에 어떤 파일들이 있고,
각각 어떤 역할을 하는지 설명해줘.
```

```
대시보드 페이지가 어떤 컴포넌트들로 구성되어 있는지
계층 구조로 보여줘.
```

```
data/mock/dashboard.json 파일을 읽어서
각 필드가 어떤 의미인지 설명해줘.
```

### 2. UI 추가

```
[위치]에 [컴포넌트]를 추가해줘.
[세부 요구사항]
```

**예시:**
```
대시보드 페이지의 AI 브리핑 패널 아래에
"카테고리별 위험도 요약" 카드를 추가해줘.
- 5대 카테고리를 가로로 나열
- 각 카테고리마다 이름, 점수, 등급 색상 표시
- 점수가 높은 순서로 정렬
- globals.css의 카테고리별 색상 변수 사용
```

### 3. UI 수정

```
[컴포넌트]의 [속성]을 [값]으로 변경해줘.
```

**예시:**
```
카테고리 리스크 바의 프로그레스 바 높이를
현재보다 2배로 키워줘.
```

```
뉴스 카드의 레이아웃을 변경해줘.
현재 세로 배치를 가로 배치로 바꿔서
왼쪽에 카테고리/등급 정보, 오른쪽에 제목/요약이 오게 해줘.
```

### 4. 데이터 수정

```
[JSON 파일]의 [필드]를 [값]으로 변경해줘.
```

**예시:**
```
data/mock/dashboard.json의 종합 점수(overallScore)를
67에서 82로 변경해줘.
```

```
data/mock/signals.json에 새로운 위기 신호를 추가해줘.
- 제목: "아파트 월세 전환 가속"
- 등급: warning
- 카테고리: realEstate
- 점수: 68
나머지 필드는 기존 데이터 형식에 맞게 채워줘.
```

### 5. API 연결

```
[API Route 파일]에서 [목업/실제] API를 호출하도록 변경해줘.
```

**예시:**
```
app/api/chat/route.ts에서
현재 목업 응답 대신 실제 Claude API를 호출하도록 변경해줘.
시스템 프롬프트는 다음과 같이 설정해줘:
"당신은 민생 위기 분석 전문가입니다.
한국어로 친절하게 답변해주세요."
```

### 6. 에러 수정 (디버깅)

```
[에러 설명]. 원인을 찾아 수정해줘.
```

**예시:**
```
대시보드 탭에서 카테고리 점수가 'undefined'로 표시돼.
원인을 찾아 수정해줘.
```

```
다음 에러가 발생했어:
"TypeError: Cannot read properties of undefined (reading 'score')"
원인을 분석하고 수정해줘.
```

```
npm run dev 실행 시 다음 에러가 나와:
[에러 메시지 전체 복사+붙여넣기]
해결해줘.
```

### 7. 기능 추가

```
[기능]을 추가해줘. 요구사항:
1. [요구사항 1]
2. [요구사항 2]
3. [요구사항 3]
```

**예시:**
```
뉴스 분석 탭에 "북마크" 기능을 추가해줘.
요구사항:
1. 각 뉴스 카드 오른쪽 상단에 북마크 아이콘 버튼
2. 클릭하면 북마크 추가/해제 토글
3. 북마크된 기사만 필터링하는 버튼
4. 북마크 상태는 브라우저 새로고침 해도 유지 (localStorage)
```

### 8. 스타일/디자인 변경

```
[컴포넌트]의 디자인을 변경해줘.
[구체적인 스타일 지시]
```

**예시:**
```
신호 카드의 디자인을 변경해줘.
- 테두리를 등급 색상으로 (왼쪽 4px 두꺼운 테두리)
- 배경에 등급 색상을 아주 연하게 적용
- 카드 사이 간격을 현재보다 넓게
- 마우스 올리면 약간 위로 올라오는 효과
- globals.css의 색상 변수를 사용해줘
```

### 9. 전체 점검

```
모든 페이지를 확인하고 에러나 문제가 있으면 알려줘.
```

```
3개 탭(대시보드, 위기신호, 뉴스분석)이 모두
정상적으로 동작하는지 확인해줘.
데이터가 올바르게 표시되는지도 점검해줘.
```

## 프롬프트 레벨업 팁

### 팁 1: 참조 지시

기존 파일이나 컴포넌트를 참고하라고 알려주면 일관된 결과를 얻을 수 있습니다.

```
data/mock/signals.json의 기존 데이터 형식을 참고해서
새로운 위기 신호 3건을 추가해줘.
```

```
components/dashboard/signal-preview-card.tsx의 스타일을 참고해서
비슷한 형태의 정책 카드를 만들어줘.
```

### 팁 2: 제약 조건 명시

프로젝트의 규칙을 함께 알려주면 규칙을 지키는 코드가 생성됩니다.

```
카드를 추가해줘. 단, 다음 규칙을 지켜줘:
- 색상은 globals.css의 CSS 변수만 사용
- 아이콘은 HugeIcons 라이브러리 사용
- 이모지 사용하지 않기
```

### 팁 3: 결과 미리 설명

원하는 최종 결과를 구체적으로 설명하면 정확도가 높아집니다.

```
최종적으로 이런 모습이어야 해:
- 화면 상단에 노란색 배너가 나타남
- 배너 안에 "주의: 물가 지수가 72점으로 상승 중입니다" 텍스트
- 오른쪽에 X 버튼을 누르면 배너가 사라짐
```

### 팁 4: 단계적 접근

큰 기능은 작은 단계로 나눠서 요청합니다.

```
1단계: "먼저 빈 카드 컴포넌트를 만들어줘"
   -> 결과 확인

2단계: "카드 안에 제목과 점수를 표시해줘"
   -> 결과 확인

3단계: "점수에 따라 색상이 바뀌게 해줘"
   -> 결과 확인

4단계: "카드를 대시보드 페이지에 배치해줘"
   -> 최종 확인
```

## 자주 쓰는 프롬프트 모음

### 개발 서버 관련
```
"npm run dev 실행해줘"
"개발 서버 재시작해줘"
"빌드 에러를 확인하고 수정해줘"
```

### 파일 관련
```
"[파일경로]를 읽어서 내용을 설명해줘"
"[파일경로]의 [부분]을 [내용]으로 수정해줘"
"[폴더]에 새 파일 [이름]을 만들어줘"
```

### Git 관련
```
"현재 변경사항을 보여줘"
"변경사항을 커밋해줘"
"최신 코드를 가져와줘" (git pull)
```

### 확인/테스트
```
"브라우저에서 [페이지]가 정상 동작하는지 확인해줘"
"콘솔에 에러가 있는지 확인해줘"
"[기능]이 올바르게 동작하는지 테스트해줘"
```

## 문제 해결 가이드

### 클로드 코드가 응답하지 않을 때
1. `Ctrl+C`로 중단
2. `/clear`로 대화 초기화
3. 다시 프롬프트 입력

### 예상과 다른 결과가 나왔을 때
1. 어떤 점이 다른지 구체적으로 설명
   ```
   "방금 만든 카드에서 색상이 잘못됐어.
    빨간색이어야 하는데 파란색으로 나와.
    --danger 변수를 사용해서 수정해줘."
   ```
2. 완전히 잘못된 경우: "방금 변경을 되돌려줘"

### 에러 메시지가 나왔을 때
1. 에러 메시지 전체를 복사
2. 클로드 코드에 그대로 전달
   ```
   "다음 에러가 발생했어:
    [에러 메시지 붙여넣기]
    원인을 찾아 수정해줘."
   ```

### 모르는 용어가 나왔을 때
```
"[용어]가 뭔지 쉽게 설명해줘"
"이 에러 메시지가 무슨 뜻인지 설명해줘"
```
