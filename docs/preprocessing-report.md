## 3. 전처리 파이프라인

### 3.1. 처리 흐름

```
원본 JSON (191,992건)
  |
  +--> [1] 언어 필터 (lang === "KR"만 통과)
  |
  +--> [2] 카테고리 코드 1차 필터
  |      매경 소분류 코드 -> IRMI 5대 카테고리 직접 매핑
  |      (약 45개 소분류 코드 대상)
  |
  +--> [3] 복합 코드 처리
  |      경제일반/정책 등 복합 코드 -> 키워드 기반 카테고리 추론
  |
  +--> [4] 키워드 2차 보완 필터
  |      1차에서 제외된 기사 중 제목/요약/키워드에
  |      민생 핵심어가 포함된 기사 추가 포함
  |
  +--> [5] 데이터 정제
  |      HTML 태그 제거, 불필요 필드 삭제,
  |      관련성 점수 산출
  |
  +--> [6] SQLite INSERT (배치 500건 단위 트랜잭션)
```

### 3.3. 데이터 정제

| 처리 | 설명 |
|------|------|
| HTML 태그 제거 | `article_body.body`의 `<p>`, `<br>`, `<img>` 등 모든 HTML 태그 strip |
| HTML 엔티티 변환 | `&nbsp;`, `&amp;`, `&lt;`, `&gt;`, `&quot;`, `&#39;` 디코딩 |
| 공백 정규화 | 연속 공백/줄바꿈을 단일 공백으로 치환 |
| 불필요 필드 제거 | `images`, `comments`, `share` (좋아요/댓글수) 미포함 |
| 키워드 배열 보존 | `keyword_list` 배열을 JSON 문자열로 저장 |

### 3.4. 관련성 점수 (relevance_score)

키워드 가중치 기반 로컬 스코어링. AI 분석 대상 우선순위 결정에 활용.

| 매칭 위치 | 가중치 (키워드당) |
|-----------|-------------------|
| 제목 (title) | +6 |
| 요약 (summary) | +3 |
| 키워드 목록 (keyword_list) | +2 |

최대 100점. 여러 키워드가 동시에 매칭되면 점수가 누적됨.

### 4.4. 전문 검색 (FTS5)

`articles_fts` 가상 테이블로 제목/요약/키워드에 대한 전문 검색 지원.

- 토크나이저: `unicode61` (공백 기반 분리, 한국어 지원)
- 인덱싱 대상: `title`, `summary`, `keywords`
- 자동 동기화: INSERT/UPDATE/DELETE 트리거로 FTS 인덱스 자동 갱신

## 6. 코드 구조

### 6.1. 관련 파일

```
scripts/
  preprocess-news.ts        # 전처리 실행 스크립트

lib/db/
  index.ts                  # DB 싱글턴 + 초기화
  schema.ts                 # 테이블/인덱스/FTS 스키마 SQL
  queries.ts                # 프로덕션 쿼리 함수 (필터/검색/페이지네이션/집계)
  category-map.ts           # 매경 -> IRMI 카테고리 매핑 + 키워드 스코어링

app/api/db-health/route.ts  # DB 헬스체크 API

data/irmi.db                # 생성된 SQLite DB (.gitignore 대상)
```

### 6.2. 실행 방법

```bash
# 전체 월 전처리
npx tsx scripts/preprocess-news.ts

# 특정 월만 전처리
npx tsx scripts/preprocess-news.ts --months 06,07,08
```

실행 시 기존 `data/irmi.db`는 삭제 후 새로 생성됨.

### 6.3. DB 헬스체크

```bash
curl http://localhost:3000/api/db-health
```

응답 예시:
```json
{
  "status": "ok",
  "articleCount": 52162,
  "analysisCount": 0,
  "signalCount": 0,
  "dateRange": {
    "earliest": "2025-01-01 06:02:48",
    "latest": "2025-12-31 23:56:50"
  }
}
```

## 7. 후속 작업 (AI 분석 파이프라인)

현재 `analysis`, `signals`, `signal_articles` 테이블은 비어 있으며, 다음 단계에서 채워짐:

1. **`scripts/analyze-batch.ts`** (작성 예정)
   - `articles` 테이블에서 관련성 점수 상위 기사를 배치로 선별
   - Claude API로 위기 관련성 분석 (제목+요약 기반 경량 분석)
   - 결과를 `analysis` 테이블에 INSERT

2. **`scripts/generate-signals.ts`** (작성 예정)
   - 분석 결과를 기반으로 위기 신호 클러스터링
   - `signals` 및 `signal_articles` 테이블 생성

3. **`scripts/generate-dashboard.ts`** (작성 예정)
   - 종합 리스크 점수, 카테고리별 위험도 등 집계
   - `dashboard_cache` 테이블에 저장

## 8. 주의사항

- `data/irmi.db`는 `.gitignore` 대상이며 원본 데이터(`data/2025/`)도 마찬가지
- 전처리 스크립트 재실행 시 기존 DB를 삭제하고 새로 생성하므로 analysis/signals 데이터도 초기화됨
- `region` 필드는 현재 `null`이며, AI 분석 단계에서 기사 본문으로부터 추출 예정
- FTS5 토크나이저(`unicode61`)는 공백 기반 분리이므로 붙여쓰기 한국어 검색에 한계 있음. `LIKE '%keyword%'` 쿼리로 보완 가능
