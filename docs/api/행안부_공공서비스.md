# 행안부 대한민국 공공서비스(혜택) 정보 API

> 보조금24 기반 정부 지원정책 데이터를 제공하는 공공데이터포털 오픈API

## 1. 개요

| 항목 | 내용 |
|------|------|
| API명 | 행정안전부_대한민국 공공서비스(혜택) 정보 |
| 제공기관 | 행정안전부 행정서비스통합추진단 |
| 데이터 출처 | 보조금24 (gov.kr) |
| 포맷 | JSON, XML |
| 인증 | 공공데이터포털 API 인증키 (serviceKey) |
| 트래픽 | 개발계정 10,000건/일 (자동승인) |
| 비용 | 무료 |
| 문의 | 1566-0025 / opendata_help@nia.or.kr |

**참조 링크:**

- 공공데이터포털: https://www.data.go.kr/data/15113968/openapi.do
- Swagger 명세: https://infuser.odcloud.kr/api/stages/44436/api-docs
- 정부24 오픈API: https://www.gov.kr/openapi

---

## 2. 인증

### Base URL

```
https://api.odcloud.kr/api
```

### 인증 방식 (택 1)

**방식 A - 쿼리 파라미터:**

```
GET /api/gov24/v3/serviceList?serviceKey={인코딩된_API키}
```

**방식 B - Authorization 헤더:**

```
GET /api/gov24/v3/serviceList
Authorization: Infuser {API키}
```

### API 키 설정

```env
# .env.local
DATA_GO_KR_API_KEY=발급받은_인증키
```

> 공공데이터포털(data.go.kr) 회원가입 후 활용신청 -> 자동승인 -> 약 30분 후 사용 가능

## 3. 엔드포인트 상세

### 3.1 공공서비스 목록 조회

정부 부처, 지자체, 공공기관, 교육청이 제공하는 수혜서비스 목록을 조회한다.

```
GET /gov24/v3/serviceList
```

#### 요청 파라미터

| 파라미터 | 위치 | 타입 | 필수 | 기본값 | 설명 |
|---------|------|------|------|-------|------|
| serviceKey | query | string | O | - | API 인증키 |
| page | query | integer | X | 1 | 페이지 번호 |
| perPage | query | integer | X | 10 | 페이지당 건수 (최대 500) |
| returnType | query | string | X | JSON | 응답 포맷 (JSON / XML) |
| cond[서비스명::LIKE] | query | string | X | - | 서비스명 포함 검색 |
| cond[소관기관유형::LIKE] | query | string | X | - | 소관기관유형 검색 |
| cond[사용자구분::LIKE] | query | string | X | - | 사용자구분 검색 |
| cond[서비스분야::LIKE] | query | string | X | - | 서비스분야 검색 |
| cond[등록일시::GTE] | query | string | X | - | 등록일시 이후 (>=) |
| cond[등록일시::LTE] | query | string | X | - | 등록일시 이전 (<=) |
| cond[수정일시::GTE] | query | string | X | - | 수정일시 이후 (>=) |
| cond[수정일시::LTE] | query | string | X | - | 수정일시 이전 (<=) |

> 날짜 필터 연산자: LT(<), LTE(<=), GT(>), GTE(>=)

## 5. IRMI 프로젝트 활용 방안

### 5.1 카테고리 매핑

IRMI의 5대 민생 카테고리와 보조금24 데이터를 다음과 같이 매핑한다.

| IRMI 카테고리 | 매핑 전략 |
|--------------|----------|
| 물가 (prices) | 서비스분야: 생활안정 / 서비스명 키워드: 물가, 생활비, 공공요금 |
| 고용 (employment) | JA0326(근로자), JA0327(구직자/실업자) / 서비스분야: 고용 |
| 자영업 (selfEmployed) | JA1101~JA1103(사업자), JA1201(음식점) / 서비스명: 소상공인 |
| 금융 (finance) | 서비스명 키워드: 금융, 대출, 이자 / 서비스분야: 서민금융 |
| 부동산 (realEstate) | JA0412(무주택) / 서비스분야: 주거/자립 / 서비스명: 주거, 전세 |

### 5.3 구현 예시 (Next.js API Route)

```typescript
// lib/api/gov-service.ts
const BASE_URL = "https://api.odcloud.kr/api/gov24/v3";

interface GovServiceListResponse {
  page: number;
  perPage: number;
  totalCount: number;
  currentCount: number;
  matchCount: number;
  data: GovService[];
}

interface GovService {
  서비스ID: string;
  서비스명: string;
  서비스목적요약: string;
  지원유형: string;
  지원대상: string;
  지원내용: string;
  신청방법: string;
  신청기한: string;
  상세조회URL: string;
  소관기관명: string;
  전화문의: string;
  서비스분야: string;
}

export async function fetchGovServices(params: {
  keyword?: string;
  field?: string;
  page?: number;
  perPage?: number;
}): Promise<GovServiceListResponse> {
  const apiKey = process.env.DATA_GO_KR_API_KEY;
  if (!apiKey) throw new Error("DATA_GO_KR_API_KEY not configured");

  const url = new URL(`${BASE_URL}/serviceList`);
  url.searchParams.set("serviceKey", apiKey);
  url.searchParams.set("page", String(params.page ?? 1));
  url.searchParams.set("perPage", String(params.perPage ?? 10));
  url.searchParams.set("returnType", "JSON");

  if (params.keyword) {
    url.searchParams.set("cond[서비스명::LIKE]", params.keyword);
  }
  if (params.field) {
    url.searchParams.set("cond[서비스분야::LIKE]", params.field);
  }

  const res = await fetch(url.toString(), { next: { revalidate: 3600 } });
  if (!res.ok) throw new Error(`API error: ${res.status}`);
  return res.json();
}
```

```typescript
// app/api/gov-services/route.ts
import { NextRequest, NextResponse } from "next/server";
import { fetchGovServices } from "@/lib/api/gov-service";

export async function GET(req: NextRequest) {
  const { searchParams } = req.nextUrl;
  const keyword = searchParams.get("keyword") ?? undefined;
  const field = searchParams.get("field") ?? undefined;
  const page = Number(searchParams.get("page") ?? 1);
  const perPage = Number(searchParams.get("perPage") ?? 10);

  try {
    const data = await fetchGovServices({ keyword, field, page, perPage });
    return NextResponse.json({ success: true, data });
  } catch (error) {
    const message = error instanceof Error ? error.message : "Unknown error";
    return NextResponse.json({ success: false, error: message }, { status: 500 });
  }
}
```

## 6. 주의사항

1. **인코딩**: 쿼리 파라미터의 한글 조건 필드명은 URL 인코딩 필요 (예: `서비스명` -> `%EC%84%9C%EB%B9%84%EC%8A%A4%EB%AA%85`)
2. **트래픽 제한**: 개발계정 10,000건/일. 운영계정은 활용사례 등록 후 증가 가능
3. **serviceDetail 조건검색 미지원**: 상세 조회는 cond 파라미터를 지원하지 않음. serviceList에서 서비스ID를 먼저 확인한 후 supportConditions에서 조건 조회
4. **캐싱 권장**: 데이터 갱신 주기가 길므로 `revalidate: 3600` (1시간) 이상 캐싱 권장
5. **서버 전용**: API 키는 서버에서만 사용. 클라이언트 직접 호출 금지
6. **2025.06 변경**: serviceDetail에 `공무원확인구비서류`, `본인확인필요구비서류` 필드 추가
