# 매경 뉴스 데이터 구조 분석서

> 매경미디어 x 앤스로픽 '뉴스 투 액션'(News to Action) AI 해커톤 제공 데이터 기반

---

## 1. 개요

### 1.1. 데이터 출처

매경미디어에서 실제 발행한 2025년 전체 기사 원문 데이터. 기사 1건당 1개의 JSON 파일로 월별 분할 제공.

### 1.2. 제공 파일 구성

| 번호 | 파일/폴더 | 설명 |
|------|-----------|------|
| 1 | 매경 뉴스 데이터 (월별) | 기사 1건 = 1 JSON 파일. 메타정보, 제목, 본문, 요약, 카테고리, 이미지, 댓글 포함 |
| 2 | News Content Format 명세서 | JSON 필드 구조 가이드 문서 |
| 3 | 매경 뉴스 카테고리.json | 전체 분류 체계 마스터 데이터 (대/중/소분류) |
| 4 | 카테고리 코드 명세서 | 카테고리 JSON 필드 구조 가이드 문서 |

### 1.3. 기본 사항

- 인코딩: **UTF-8**
- 결측치: 문자열 필드 -> `null` 또는 `""` (빈 문자열), 리스트형 필드 -> `[]` (빈 배열)
- 보안: `.gitignore` 처리 필수 (데이터 파일 공개 금지)

## 2. 파일 구조

### 2.1. 파일명 규칙

```
hackathon_{Action}_{Timestamp}_{article_id}.json
```

| 요소 | 설명 | 예시 |
|------|------|------|
| Action | 데이터 처리 액션. I: 입력, U: 수정, D: 삭제 (해커톤은 I만 제공) | `I` |
| Timestamp | 파일 생성 시점 (밀리초 포함, 17자리) | `20260213143025123` |
| article_id | 기사 고유 식별 번호 | `10509708` |

예시: `hackathon_I_20260213143025123_10509708.json`

### 2.2. JSON 최상위 구조

```json
{
  "article": { ... },          // 기사 기본정보 (메타데이터)
  "article_body": { ... },     // 기사 본문
  "article_summary": { ... },  // 기사 요약
  "article_url": "...",        // 기사 웹 URL
  "categories": [ ... ],       // 분류 정보 (배열)
  "keyword_list": [ ... ],     // 키워드 목록 (배열)
  "images": [ ... ],           // 이미지 정보 (배열)
  "share": { ... },            // 반응 통계 (좋아요/댓글 수)
  "comments": [ ... ]          // 댓글 목록 (배열)
}
```

## 3. 기사 데이터 상세 필드 정의

### 3.1. article (기사 기본정보)

기사의 메타데이터 및 기본 속성.

| 필드명 | 타입 | 설명 | 예시 |
|--------|------|------|------|
| `article_id` | Number | 기사 고유 식별자 (ID) | `10509708` |
| `service_daytime` | String | 기사 출고(서비스) 일시 | `"2026-02-13 09:30:00"` |
| `title` | String | 기사 제목 | `"코스피, 외국인 매수에 2,650선 돌파"` |
| `sub_title` | String | 기사 부제목 | `"삼성전자 동반 상승세"` |
| `keywords` | String | 키워드 전체 문자열 (쉼표 구분) | `"코스피,반도체,삼성전자"` |
| `writers` | String | 기자 이름(이메일) 형식 | `"박윤균 기자(gyun@mk.co.kr)"` |
| `lang` | String | 기사 작성 언어 | `"KR"` (한국어), `"EN"` (영어) |
| `pub_div` | String | 출고 유형 | `"W"` (웹 기사), `"P"` (지면 기사) |
| `pub_date` | String | 지면 게재일 (yyyyMMdd) | `"20260213"` |
| `pub_section` | String | 신문 부 번호 | `"01"` (첫번째 부) |
| `pub_page` | Number/null | 지면 게재 면수 (웹 전용은 null) | `1` |
| `reg_dt` | String | 기사 최초 등록 일시 | `"2026-02-13 08:45:00"` |
| `mod_dt` | String | 기사 최종 수정 일시 | `"2026-02-13 09:25:00"` |
| `main_category` | String | 대표 분류 코드 | `"MK100101"` |

### 3.2. article_body (기사 본문)

| 필드명 | 타입 | 설명 |
|--------|------|------|
| `body` | String | **HTML 태그가 포함된** 기사 본문 전체 |

> 주의: 본문에 HTML 태그(`<p>`, `<br>`, `<img>` 등)가 포함되어 있어 텍스트 추출 시 태그 제거(strip) 처리가 필요함.

### 3.3. article_summary (기사 요약)

| 필드명 | 타입 | 설명 |
|--------|------|------|
| `summary` | String | 기사 요약 텍스트 |

> 참고: 실제 요약본이 아닌 본문 앞부분을 단순 잘라낸 경우가 있음.

### 3.4. article_url (기사 링크)

| 필드명 | 타입 | 설명 | 예시 |
|--------|------|------|------|
| `article_url` | String | 기사 웹 페이지 전체 URL | `"https://www.mk.co.kr/news/stock/10509708"` |

### 3.5. categories (분류 정보)

기사가 속한 분류 체계 정보. **배열 형태**로 복수 카테고리 가능.

| 필드명 | 타입 | 설명 | 예시 |
|--------|------|------|------|
| `code_id` | String | 최종 분류 코드 | `"MK300113"` |
| `code_nm` | String | 최종 분류명 | `"Consumer Journal"` |
| `large_code_id` | String | 대분류 코드 | `"0"` |
| `large_code_nm` | String | 대분류명 | `"뉴스"` |
| `middle_code_id` | String | 중분류 코드 | `"00301"` |
| `middle_code_nm` | String | 중분류명 | `"경제"` |
| `small_code_id` | String | 소분류 코드 | `"MK300113"` |
| `small_code_nm` | String | 소분류명 | `"Consumer Journal"` |

> 참고: `code_id`/`code_nm`은 최종 적용 분류 코드로, 대부분 `small_code_id`/`small_code_nm`과 동일하지만 다를 수 있음.

### 3.6. keyword_list (키워드 목록)

| 타입 | 설명 | 예시 |
|------|------|------|
| Array\<String\> | 키워드를 개별 문자열로 분리한 배열 | `["코스피", "반도체", "삼성전자"]` |

> `article.keywords`는 쉼표 구분 문자열, `keyword_list`는 분리된 배열 형태로 동일 데이터의 두 가지 표현.

### 3.7. images (이미지 정보)

기사에 포함된 이미지 배열. 이미지 없으면 `[]`.

| 필드명 | 타입 | 설명 | 예시 |
|--------|------|------|------|
| `image_url` | String | 이미지 원본 URL | `"https://wimg.mk.co.kr/news/cms/..."` |
| `image_caption` | String | 이미지 설명(캡션) | `"유한양행"` |

### 3.8. share (반응 통계)

| 필드명 | 타입 | 설명 |
|--------|------|------|
| `like_count` | Number | 좋아요 등 긍정적 반응 수 |
| `reply_count` | Number | 전체 댓글 수 (대댓글 포함) |

### 3.9. comments (댓글 목록)

노출 가능한 상태의 댓글. 삭제/비공개 댓글 제외. 댓글 없으면 `[]`.

| 필드명 | 타입 | 설명 | 예시 |
|--------|------|------|------|
| `comment_id` | Number | 댓글 고유 번호 | `813501` |
| `parent_id` | Number | 부모 댓글 번호 (0=원본, 그 외=대댓글) | `0` |
| `author` | String | 작성자명 (앞 3글자 이후 `*` 처리) | `"abc****"` |
| `content` | String | 댓글 내용 | `"오늘 반도체 분위기 좋네요!"` |
| `like_count` | Number | 댓글 좋아요 수 | `15` |
| `hate_count` | Number | 댓글 싫어요 수 | `0` |
| `created_at` | String | 작성 일시 (yyyy-MM-dd HH:mm:ss) | `"2026-02-13 10:05:23"` |

## 4. 카테고리 코드 체계

### 4.1. 구조 개요

매경 뉴스 카테고리는 **3단계 계층 구조**: 대분류 > 중분류 > 소분류.

카테고리 마스터 JSON 구조:
```json
{
  "_DATA": [ { ... }, { ... } ],       // 분류 코드 상세 목록
  "_CREATE_TIME": "20260219 12:29:32.422",  // 생성 시점
  "_WORK_TIME": 11                          // 추출 소요 시간(ms)
}
```

`_DATA` 각 항목 필드 (Key는 **대문자**):

| 필드명 | 타입 | 설명 |
|--------|------|------|
| `LARGE_CODE_ID` | String | 대분류 코드 ID |
| `LARGE_CODE_NM` | String | 대분류 코드명 |
| `MIDDLE_CODE_ID` | String | 중분류 코드 ID |
| `MIDDLE_CODE_NM` | String | 중분류 코드명 |
| `SMALL_CODE_ID` | String | 소분류 코드 ID (기사 매핑 시 주로 사용) |
| `SMALL_CODE_NM` | String | 소분류 코드명 |
| `SEQ` | Number | 코드 정렬 순서 |

### 5.2. 데이터 처리 시 유의사항

1. **main_category vs categories 배열**: `article.main_category`는 대표 카테고리 1개, `categories`는 복수 분류 가능. 1차 필터링은 `main_category`, 상세 분류는 `categories` 배열 활용.

2. **키워드 중복 표현**: `article.keywords`(쉼표 구분 문자열)와 `keyword_list`(배열)는 동일 데이터. 배열인 `keyword_list` 사용 권장.

3. **HTML 본문 처리**: `article_body.body`에 HTML 태그 포함. 텍스트 분석 시 태그 제거(strip tags) 필요. AI 분석 시에도 HTML 제거 후 전달 권장.

4. **요약 데이터 품질**: `article_summary.summary`가 실제 요약이 아닌 본문 앞부분만 자른 경우 있음. AI를 통한 재요약 고려.

5. **시간 필드 형식 차이**:
   - `service_daytime`, `reg_dt`, `mod_dt`: `yyyy-MM-dd HH:mm:ss` 형식
   - `pub_date`: `yyyyMMdd` 형식 (날짜만)
   - 파일명 Timestamp: `yyyyMMddHHmmssSSS` 형식 (17자리)

6. **null/빈값 처리**: `pub_page`은 웹 전용 기사일 때 `null`. 문자열 필드는 빈 문자열(`""`) 가능. `images`, `comments`는 빈 배열(`[]`) 가능.
